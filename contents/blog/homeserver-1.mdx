# 홈서버 구축기#1

## 우분투 설치하기

홈서버의 첫 걸음은 OS 설치다. 필자는 안정성과 커뮤니티 지원이 좋은 우분투 22.04 LTS를 선택했다.

1. **우분투 ISO 다운로드**: 우분투 공식 사이트에서 22.04 LTS 버전 ISO를 받는다.
2. **부팅 USB 만들기**: Rufus나 Etcher로 부팅 USB를 만든다.
3. **BIOS 설정**: 서버로 쓸 컴퓨터의 BIOS에 들어가 부팅 순서를 USB 먼저로 바꾼다.
4. **설치 시작**: USB로 부팅하고 'Install Ubuntu Server'를 선택한다.
5. **언어 및 키보드 설정**: 편한 언어로 설정한다. 필자는 영어로 했다.
6. **네트워크 설정**: DHCP를 쓰거나 고정 IP를 설정한다.
7. **저장소 설정**: 전체 디스크를 사용할 건지, 파티션을 나눌 건지 선택한다.
8. **계정 설정**: 루트 계정과 일반 사용자 계정을 만든다.
9. **SSH 설정**: OpenSSH server 설치를 선택한다.
10. **설치 완료**: 재부팅하고 USB를 제거한다.

이렇게 하면 기본적인 우분투 서버가 설치된다. 이제 보안 설정을 해보자.

## 고급 보안 설정

서버를 인터넷에 연결하기 전에 꼭 해야 할 보안 설정들이 있다. 필자는 이것들을 하나씩 적용해봤다.

### 1. 업데이트 및 업그레이드

가장 먼저 할 일은 시스템을 최신 상태로 만드는 거다.

```bash
sudo apt update
sudo apt upgrade -y
```

### 2. SSH 보안 강화

SSH는 원격 접속의 기본이지만, 그만큼 공격 대상이 되기 쉽다.

```bash
sudo nano /etc/ssh/sshd_config
```

이 파일에서 다음 설정들을 변경하자:

- `PermitRootLogin no`: 루트 로그인 차단
- `PasswordAuthentication no`: 비밀번호 인증 비활성화
- `Port 22222`: SSH 포트 변경 (22는 너무 뻔하다)

변경 후 SSH 서비스를 재시작한다:

```bash
sudo systemctl restart sshd
```

### 3. iptables 설정

UFW 대신 iptables를 사용해 더 세밀한 방화벽 설정을 해보자.

```bash
# 기본 정책 설정
sudo iptables -P INPUT DROP
sudo iptables -P FORWARD DROP
sudo iptables -P OUTPUT ACCEPT

# 로컬호스트 통신 허용
sudo iptables -A INPUT -i lo -j ACCEPT
sudo iptables -A OUTPUT -o lo -j ACCEPT

# 설정한 SSH 포트 허용
sudo iptables -A INPUT -p tcp --dport 22222 -j ACCEPT

# 설정 저장
sudo apt install iptables-persistent
sudo netfilter-persistent save
```

### 4. Fail2Ban 설치

무차별 대입 공격을 막아주는 Fail2Ban을 설치하자.

```bash
sudo apt install fail2ban
sudo cp /etc/fail2ban/jail.conf /etc/fail2ban/jail.local
sudo nano /etc/fail2ban/jail.local
```

`jail.local` 파일에서 SSH 설정을 찾아 다음과 같이 수정한다:

```
[sshd]
enabled = true
port = 22222
filter = sshd
logpath = /var/log/auth.log
maxretry = 3
bantime = 3600
```

설정 후 Fail2Ban을 재시작한다:

```bash
sudo systemctl restart fail2ban
```

### 5. 자동 업데이트 설정

보안 패치를 자동으로 적용하도록 설정하자.

```bash
sudo apt install unattended-upgrades
sudo dpkg-reconfigure -plow unattended-upgrades
```

### 6. 로그인 배너 설정

서버에 접속하려는 사람에게 경고 메시지를 보여주자.

```bash
sudo nano /etc/issue.net
```

파일에 다음과 같은 내용을 추가한다:

```
*******************************************************************
                            경고!
이 시스템은 권한이 있는 사용자만 접근할 수 있습니다.
모든 활동은 모니터링되고 기록됩니다.
무단 접근 시도는 법적 처벌을 받을 수 있습니다.
*******************************************************************
```

그리고 SSH 설정 파일을 수정한다:

```bash
sudo nano /etc/ssh/sshd_config
```

다음 줄을 추가 또는 수정한다:

```
Banner /etc/issue.net
```

SSH 서비스를 재시작한다:

```bash
sudo systemctl restart sshd
```

### 7. 루트 계정 잠그기

루트 계정을 완전히 잠가버리자.

```bash
sudo passwd -l root
```

### 8. 불필요한 서비스 제거

사용하지 않는 서비스는 제거하는 게 좋다.

```bash
sudo apt remove telnet
sudo apt autoremove
```

### 9. 파일 시스템 보안 강화

중요한 파일들의 권한을 제한하자.

```bash
sudo chmod 700 /etc/shadow
sudo chmod 700 /etc/gshadow
sudo chmod 644 /etc/group
sudo chmod 644 /etc/passwd
```

### 10. 네트워크 보안 강화

IP 스푸핑과 같은 공격을 방지하기 위해 커널 파라미터를 수정하자.

```bash
sudo nano /etc/sysctl.conf
```

다음 내용을 추가한다:

```
net.ipv4.conf.default.rp_filter=1
net.ipv4.conf.all.rp_filter=1
net.ipv4.conf.all.accept_redirects=0
net.ipv6.conf.all.accept_redirects=0
net.ipv4.conf.all.send_redirects=0
net.ipv4.conf.all.accept_source_route=0
net.ipv6.conf.all.accept_source_route=0
```

변경사항을 적용한다:

```bash
sudo sysctl -p
```

## 마치며

이렇게 우분투 설치와 고급 보안 설정을 완료했다. 솔직히 말하면, 이 정도만 해도 꽤 안전한 편이다. 하지만 보안은 끝이 없는 영역이라 계속 공부하고 업데이트해야 한다.

다음 글에서는 Docker를 설치하고 컨테이너를 운영하는 방법에 대해 다뤄볼 예정이다. Docker를 쓰면 서비스별로 환경을 깔끔하게 분리할 수 있어서 관리가 편해진다. 기대해도 좋다!

마지막으로 한마디. 보안 설정할 때 너무 겁먹지 말자. 천천히 하나씩 해나가다 보면 어느새 꽤 안전한 서버가 되어 있을 거다. 그리고 실수해도 괜찮다. 어차피 집에서 노는 서버니까 망가뜨려도 다시 설치하면 그만이다. 이런 시행착오가 다 공부가 되는 거다.

자, 이제 필자의 홈서버가 인터넷의 험한 바다에서 살아남을 준비가 되었다. 다음 여정을 향해 출발!
